基于STM32的仓储环境检测系统
项目简介
本系统针对中小型粮仓在粮食储存过程中存在的监测方法落后、成本高昂及智能化水平不足等问题，设计并实现了一套基于STM32的仓储环境检测系统。系统集成数据采集、处理、显示与远程通信功能，可实时监测粮仓内的温度、湿度、光照、烟雾浓度及粮食含水率等关键参数，并支持本地报警、自动控制及云端远程监控。

主要功能
环境参数采集：使用DHT11（温湿度）、DS18B20（粮温）、YL-69（粮食湿度）、光敏电阻（光照强度）、MQ-2（烟雾浓度）等多种传感器。

数据处理与算法：引入EMC（平衡水分）算法，通过环境温湿度间接估算粮食含水率。

本地显示：OLED屏幕实时显示各项环境数据。

自动控制：当参数超限时，自动触发蜂鸣器报警、启动风扇通风。

远程监控：通过ESP8266模块连接Wi-Fi，采用MQTT协议上传数据至ONENET云平台，用户可通过上位机（Qt5开发）或云平台实时查看数据并下发控制指令。

软件设计
下位机（STM32）：使用Keil MDK开发，C语言编写。包含传感器驱动、ADC采集、单总线协议、I2C驱动、串口通信、MQTT封装、EMC算法等。

上位机：基于Python + Qt5开发，通过HTTP协议与ONENET云平台交互，实现数据可视化与远程控制。

云平台：中国移动ONENET，采用MQTT协议接收设备数据，并存储历史数据。

使用说明
硬件连接
按照论文中电路图连接各模块，确保供电电压正确（STM32及传感器3.3V/5V兼容）。

下位机烧录
使用串口或SWD方式将程序烧录至STM32F103C8T6。

Wi-Fi配置
修改程序中ESP8266连接的SSID和密码，以及ONENET平台的产品ID、设备名和Token。

云平台配置
在ONENET创建产品与设备，获取MQTT连接参数。

运行
上电后系统自动初始化，OLED显示数据。登录ONENET或打开上位机即可远程监测。

测试结果
传感器数据采集准确，响应及时。

ESP8266通信稳定，数据上传与指令下发延迟较小。

EMC算法反推含水率误差控制在0.5%以内。

自动控制逻辑正确：超限时蜂鸣器报警、风扇启动。

上位机能正常显示实时数据并下发控制命令。
